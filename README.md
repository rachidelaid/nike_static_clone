# Nike Static Clone

A high-performance, responsive athletic retail site inspired by premium Nike-style ecommerce experiences. The project uses a Stitch-led design workflow, React components, and Tailwind CSS to turn generated product screens into a static frontend.

## Live Demo

Check out the deployed site here: [Nike Static Clone](https://nike-static-clone-8a7k28wsx-rachid-elaids-projects.vercel.app/)

## Skills Used In Order

1. **stitch-design:** Generated the high-fidelity APEX FORM retail screens and established the direction for the product experience.
2. **design-md:** Synthesized the Stitch output into `.stitch/DESIGN.md`, documenting the visual theme, tokens, typography, components, and layout rules.
3. **react:components:** Converted the Stitch screen direction into modular React components for the Vite frontend.
4. **download-images:** Localized remote/generated visual assets so the app can render reliably from project files.
5. **OpenCode with GPT-5.5:** Assisted with implementation, refinement, validation, and documentation updates.

## Tech Stack

- **Framework:** [React](https://react.dev/)
- **Routing:** [TanStack Router](https://tanstack.com/router)
- **Styling:** [Tailwind CSS](https://tailwindcss.com/)
- **Tooling:** [Vite+](https://viteplus.dev/guide/)
- **Deployment:** [Vercel](https://vercel.com/)

## Features

- **Responsive design:** Optimized for mobile, tablet, and desktop views.
- **Premium retail visuals:** Warm editorial surfaces, oversized type, product-first imagery, and high-contrast commerce moments.
- **Component-based architecture:** Modular React components for maintainable page composition.
- **Local Stitch references:** Design previews and system guidance live in `.stitch/` alongside the app source.

## Installation & Setup

1. **Clone the repository:**

   ```bash
   git clone https://github.com/your-username/nike-clone.git
   ```

2. **Navigate to the directory:**

   ```bash
   cd nike-clone
   ```

3. **Install dependencies:**

   ```bash
   vp install
   ```

4. **Run the development server:**

   ```bash
   vp dev
   ```

5. **Validate changes:**

   ```bash
   vp check
   vp test
   ```
